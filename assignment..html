Praksisprosjekt IS-302 H25

Lag en fullstendig responsiv, moderne nettside som demonstrerer din forståelse av HTML, CSS og webdesignprinsipper. Dette prosjektet vil vise din evne til å bygge profesjonelle, brukersentriske webopplevelser.

Prosjektmål

  • Utvikle en responsiv nettside som fungerer på alle enheter
  • Implementere moderne CSS-teknikker og best practices
  • Sikre god brukeropplevelse og tilgjengelighet
  • Dokumentere utviklingsprosessen grundig

Detaljert beskrivelse

Tekniske krav

Nettsiden skal bygges med semantisk HTML5 og moderne CSS3. Responsive design er obligatorisk, og siden skal fungere optimalt på mobile enheter, nettbrett og desktop-datamaskiner.

Funksjonelle krav

  • Navigasjon som fungerer på alle sider
  • Kontaktskjema med validering
  • Bildegalleri eller portfolio-seksjon
  • Footer med relevant informasjon

Leveranser

🌐

Nettside

Fullstendig funksjonell nettside med alle påkrevde sider og funksjoner

📄

Prosjektrapport som forklarer designvalg og prosess

🎥

Presentasjon

Ytterligere krav

  • GitHub-repositorium med meningsfull commit-historikk
  • Live distribusjons-URL (Netlify, Vercel eller GitHub Pages)
  • Testdokumentasjon med nettleserkompatibilitetsmatrise

Prosjekt Tidslinje

Uke 32

Prosjektoppstart og planlegging

7. august til 2. september

Innledende briefing, konseptutvikling og wireframe-opprettelse. Send inn prosjektforslag.

Uke 35

Interessent kartlegging

2. september til 16. september

Lag mockups, etabler visuelt designsystem, og samle tilbakemeldinger fra kolleger.

Uke 38

Ideutforming / MVP

23. september til 7. oktober

Bygg kjernefunksjonalitet, implementer responsive oppsett, og integrer alt innhold.

Uke 42

Testing og forbedring

14. oktober til 28. oktober

Utfør grundig testing på ulike enheter, optimaliser ytelse og rett opp feil.
